# Lesson 12: Designing UX for AI Applications

## Microsoft Course Concept

This lesson focuses on user experience in AI applications. It emphasizes that AI apps should meet user needs and be useful, reliable, accessible, and pleasant.

The lesson also highlights trust, transparency, explainability, user control, collaboration, feedback, and error handling as core design concerns.

## My Adaptation

I adapted the lesson into an AI UX Message Framework Generator for my ResearchLab AI tools.

The app generates reusable UX messages and design checks that can be added to AI applications to improve transparency, user trust, human review, feedback, and data control.

## Main Lesson

AI applications should not only produce outputs. They should clearly explain what they can do, what they cannot do, how users remain in control, when human review is needed, and how users can give feedback.

## Build 2: AI Error and Feedback Handler

I built an error and feedback handler for AI applications.

The app generates user-friendly messages for unclear input, missing data, low-confidence output, out-of-scope requests, evidence limitations, human review requirements, system errors, and feedback capture.

This supports the Lesson 12 focus on reliability, trust, transparency, collaboration, and user feedback.
